Статические методы


Как выглядит код: Что происходит на самом деле:
Cat cat = new Cat();
String name = cat.getName();
cat.setAge(17);
cat.setChildren(cat1, cat2, cat3);		 Cat cat = new Cat();
String name = cat.getName(cat);
cat.setAge(cat,17);
cat.setChildren(cat,cat1, cat2, cat3);

При вызове метода, на самом деле вызывается метод класса, первым агрументом в который передается "тот самый объект". Внутри метода он получает имя this. Именно с ним и его данными происходят все действия.

Вот как работает статический метод:

Так выглядит код: Что происходит на самом деле:
Cat cat1 = new Cat();
Cat cat2 = new Cat();
int catCount = Cat.getAllCatsCount();		 Cat cat1 = new Cat();
Cat cat2 = new Cat();
int catCount = Cat.getAllCatsCount(null);

При вызове статического метода никакого объекта внутрь не передается, то есть this равен null. Статический метод не имеет доступ к нестатическим переменным и методам. Ему нечего передать в качестве this.

Статические переменные

Статические переменные существуют в одном экземпляре и обращаться к ним нужно поп имени класса (внутри класса можно просто про имени).
