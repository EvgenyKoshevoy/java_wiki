Методы

Общий синтаксис объявления метода:

возвращаемый_тип имя (список_параметров) {
    //тело метода
}

возвращаемый_тип - тип данных, возвращаемый методом. Им может быть любой допустимый тип, в том числе и тип класса.
Если метод не возвращает значение, то для него указывают тип void.

список_параметров - последовательность разделенных запятыми параметров, для которых указывается тип и имя. Параметры представляют собой переменные, которые получают значения, передаваемые им в виде аргуметов при вызове метода. Если у метода отсутствуют параметры, список параметров оказывается пустым. 

Сигнатура метода - включает в себя название метода и типы параметров в определеном порядке.

Возврат из метода

Осуществляется двумя способами. Первый - достижение закрывающей фигурной скобки. Второй - через оператор return.

Есть две разновидности return:
1. Для методов типа void не возвращающих значение;
2. Для методов возвращающих значение.

Для типа void:

void myMeth() {
    int i;
    for (i=1; i < 10; i++) {
        if (i ==5) return; //цикл завершится  на значении 5
        System.out.println();
    }
}

Синтаксис:

void myMeth() {
    //...
    if (done) return;
    //...  
    if (error) return;
    //...
}

Возврат значения:
Синтаксис:
return значение;

Использование параметров

Аргумент - значение передаваемое методу
Параметр -  переменная, получающая аргумент. Объявляется в скобках после имени метода.

Параметров может быть несколько, типы могут быть разными:

myMeth (int a, double b, String name) {
    //...
}

Методы с переменным числом аргументов

Пример:

static void vaTest(int ... v) {
    //..
}

Может объявляться со обычными аргументами:

static void vaTest2(String msg, int ... v) {
    //..
}

Ограничеия:
- список параметров переменной длины указывается последним;
- список параметров переменной длины в методе можно указать только один раз;

Перегрузка методов

Условия:
- тип или число параметров в каждом из перегружаемых методов должны быть разными
- тип возвращаемого значения не учитывается при перегрузке

Пример:

void ovlDemo() {
    //..
}

void ovlDemo(int i) {
//...
}

int ovlDemo(int a, int b) {
    //..
}

double ovlDemo(double a, double b) {
    //..
}

Следующая попытка перегрузки скомпелируется с ошибкой, т.к. тип возвращаемого значения не учитывается при перегрузке:

void ovlDemo(int a) {
    //..
}

int ovlDemo(int a) { //компилятор неразличит два этих метода
    //..
}

Создание методов с одинаковыми названиями, но с разными параметрами.

public void sayHi(String firstGuest) {
    //...
}

public void sayHi(String firstGuest, String secondGuest) {
    //...
}

Для методов с переменной длиной аргументов:

public void sayHi(String...names) {
    for (String name: names) {
        System.out.println("Say Hello" + name);
    }
}

    В процессе определения метода, который нужно вызвать, типы могут только расширяться, но не сужаться.
  Пример:

class Cat {
    public static void print(short n) {
        System.out.println(n);
    }
    public static void print(Integer n) {
        System.out.println(n);
    }

    public static void main(String[] args) {
        Cat.print((byte) 1); //byte будет расширен до short и лишь потом до int. Будет вызван первый метод.
        Cat.print(1); //неявно будет выполнено разрешенное преобразование от int к Integer. Сработает второй метод.
    }
 }

Следует всегда указывать оператор преобразования типа, чтобы знать какой метод вызовется.


Переопределение методов

Пример:

class A {
    int i, j;

    A(int a, int b) {
        i = a;
        j = b;
    }

    void show() {
        System.out.println("i and j: " + i + " " + j);
    }

}

class B extends A {
    int k;

    B(int a, int b, int c) {
        super(a, b);
        k = c;
    }

    void show() {
        System.out.println("k: " + k);
    }

}

Если метод show() вызывается для объекта типа В, выбирается версия этого метода, определенная в классе В. Для доступа к методу суперкласса требуется использование ключевого слова super.

Пример:

class B extends A {
    int k;

    B(int a, int b, int c) {
    super(a, b);
    k = c;
    }

    void show() {
        super.show(); //вызов метода show() определенного в суперклассе
        System.out.println("k: " + k);
    }
}

Методы с разными сигнатурами не переопределяются, а перегружаются!

У переопределенного метода должен быть тот же тип возвращающемого значения, что и у метода родителя.

Модификатор доступа у переопределяемого метода должен совпадать с оригинальным.

Наследовать и переопределять можно только нестатические методы. Статические методы не наследуются и не переопределяются.

Набор методов, которые можно  вызвать у переменной, определяется типом переменной, а какая именно сработает реализация метода определяется типом объекта.

Абстрактые методы

Объявляется с использованием спецификации abstract. Такой метод не имеет тела и не реализуется в суперклассе, т.е. он должен быть переопределен в подклассе.

Синтаксис:
    abstract тип имя (список_параметров);

Спецификация abstract може тприменяться только к обычным методам (не к статическим или конструкторам).
