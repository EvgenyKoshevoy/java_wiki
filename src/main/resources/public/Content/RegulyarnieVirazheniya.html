Регулярные выражения

    Обработка регулярных выражений поддерживается классами Pattern и Matcher.

    Matcher - входная последовательность.

    Pattern - шаблон для поиска.

    Класс Pattern

    Конструкторы не определяются. Вместо этого вызывается фабричный метод compile():
        static Patter compile(String шаблон)
            шаблон - регулярное выражение, которое требуется использовать.

    Метод преобразует шаблон в символьную строку для сопоставления средствами класса Matcher. Возвращает объект типа Pattern.

    Как только объект Pattern получен, можно вызвать фабричный метод matcher() для создания объекта Matcher:
        Matcher matcher(charSequence строка)
            строка - последовательность символов сопоставляемая с шаблоном, называется входной последовательностью.

    Так же методу можно передать символьную строку.

    Класс Matcher

    Не имеет конструкторов. Для создания используется фабричный метод matcher(), описанный в классе Pattern.

    Методы:

        boolean matches() - определяет совпадает ли последовательность символов с шаблоном.
            Возварщает true если совпадает вся последовательность (для проверки совпадения подпоследовательности используется другой метод),                         иначе false.

        boolean find() - определяет совпадает ли подпоследовательность с шаблоном.
            Возвращает true, если подпоследовательность совпадает с шаблоном, иначе false. Можно вызвать неоднократно, при каждом вызове сравнение начинается с того места, где было завершено предыдущее сравнение.

        String group() - возвращает символьную строку, содержащую последнюю совпавшую последовательность. Если совпадений не обнаружено генерируется исключение IllegalStateException.

        int start() - возвращает индекс текущего совпадения во входной последовательности символов.

        int end() - возвращает индекс следующий, после текущего совпадения.
       
        String replaceAll(String новая_строка) - каждую совпавшую последовательность заменяет другой последовательность новая_строка.

   

    Синтаксис регулярных выражений

    Если регулярное выражение составлено не правильно генерируется исключение: PatternSyntaxException

    Шаблон определяется так:
        "шаблон"
        То есть если определить "xy", то с этим шаблоном может совпадать только последовательность xy.

    Символы вроде новой строки и табуляции указываются в виде \ (\n)

    Класс символов
        Является набором символов: [набор_символов]

    Обратный набор символов
        [^набор_символов]

    Диапазон символов
        [1 - 9]

    Метасимвол . - совпадает с любым символом

    Квантор определяет сколько раз совпадает выражение:   

Квантор
 Описание
+
 Обозначает совпадение один раз или больше
*
 Обозначает совпадение нуль раз или больше
?
 Обозначает совпадение нуль раз или один

    Обращение с классами символов

public class RegExpr7 {

    public static void main(String[] args) {
        Pattern pat = Pattern.compile("[a-z]+");
        Matcher mat = pat.matcher("this is a test.");
       
        while(mat.find()) {
            System.out.println("Совпадение: " + mat.group());
        }
    }
}

    Применение метода replaceAll()

// Применить метод replaceAll()
// Входная последовательность Jon заменяется выходной последовательостью Eric

public class RegExpr8 {

    public static void main(String[] args) {
        String str = "Jon Jonathan Frank Ken Todd";
       
        // Регулярное выражение Jon.*? выполняет сопоставленеи с любой символьной строкой, начинающейся с
        // Jon после которой следует нуль или больше символов
        Pattern pat = Pattern.compile("Jon.*?");
        Matcher mat = pat.matcher(str);
       
        System.out.println("Исходная последовательность: " + str);
       
        str = mat.replaceAll("Eric");
       
        System.out.println("Измененная последовательность: " + str);
    }
}

    Применение метода split()

    Синтаксис:
        String[] split(CharSequence строка)

// Применнеие метода split()

public class RegExpr9 {

    public static void main(String[] args) {
       
        Pattern pat = Pattern.compile("[ ,.!]+");
       
        String[] str = pat.split("one two, alpha9 12!done");
       
        for(int i = 0; i < str.length; i++) {
            System.out.println("Следующая лексима: " + str[i]);
        }
    }
}

    Пример : a{10, 20} - определить, содержится ли в документе от 10 до 20 букв.
   
    + - соответствует максимально возможному значению.   

    Пример: <a href="helloworld.htm" title="Привет мир!">Привет мир!</a> 

    ".*" - шаблон соответствует тексту, заключенному в двойные кавычки. Найдет следующее:
        "helloworld.htm" title="Привет мир!"

    ".*?" - за счет ? этот шаблон ищет наименьшее из возможых вхождений. Найдет следующее:
        "helloworld.htm" "Привет мир!"

     Модификаторы

    Символ "-" перед модификатором (кроме U) создае его отрицание.

g глобальный поиск, обрабатываются все совпадения с шаблоном поиска
i игнорировать регистр
m многострочный поиск
s текст восприимается как одна строка. Символ "." будет включать и перевод строки
u используется кодировка UTF
U инвертировать жадность
x игнорировать все неэкранированные пробелы и перечисленные в классе символы

    Якоря

    Указывают на начало или конец чего либо.
   
    Шаблон, чоответствующий строке, начинающейся с цифры: ^[0-9]+
        ^ - начало строки

    Утверждения

    Найти слово, включающее букву q, за которой не следует последовательность "werty": [^\s]*q(?!werty)[^\s]*

    Группы и диапазоны

    Содержит ли строка шестнадцатиричные цифры: [A-Fa-f0-9]

    Поиск ab или bc: (ab | bc)

    Поиск aaa123aaa: (aaa | bbb)[0-9]+\1
        \1 - ссылается на первую предыдущую группу (ааа)

    Найти вхождение слова wish с предыдущими и последующими символами, если это не буквы или цифры: ([^A-Za-z0-9])(wish)([^A-Za-z0-9])
