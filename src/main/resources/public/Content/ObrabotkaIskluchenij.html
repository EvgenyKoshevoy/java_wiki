Обработка исключений

    Все классы исключений являются потомками класса Throwable.

    Подклассы:
    - Exception (Ошибки, связанные с работой программы)
        - RuntimeExcteption (служит для представления различных видов ошибок)
    - Error (Ошибки, возникающий в виртуальной машине Java)

    Обработка исключений

    Способ 1:
        -с помощью блока try{} catch{}
   
    Способ 2:
        - пробросить исключение вверх по стеку вызовов
    Проброс осуществляется с помощью слова throws.
       
    Пример:
    method1() throws exception1, exception2, ..., exceptionN {
        //...
    }

    Теперь если кто-то захочет использовать method1, он должен будет обработать это исключение в своем методе при помощи try - catch, либо пробросить его дальше:

    method2() throws exception1, exception 2 {
        method1();
    }

    Ключевые слова

    Для обработки исключений предусмотрены 5 ключевых слов:

  •     try (Блок кода, в котором требуется отслеживать появление исключений)
  •     catch
  •     throw (Используется для генерирования исключения вручную)
  •     throws (Используется для обработки исключений, за пределами метода, в которых они возникают)
  •     finally (Код, который должен быть выполнен в любом случае, после выхода из блока try)

    Использование try и catch

    Используются совместно:

try {
    //блок кода, в котором должны отслеживаться ошибки
}
catch (тип_исключения1 объект_исключения){
    //обработчик исключения тип_исключения1
}

catch (тип_исключения2 объект_исключения) {
    //обработчик исключения тип_исключения2
}

    Catch выполняется только при появлении исключения.
    При возникновении исключения выполнение блока try прекращается и начинается выполнение блока catch

    Устранение программных ошибок

    Пример:

class ExcDemo3 {
    public static void main(String[] args) {
        int[] numer = {4, 8, 16 , 32, 64, 128};
        int[] denum = {2, 0, 4, 4, 0, 8};

        for (int i =0; i < numer.length; i++) {
            try{
                System.out.println(numer[i] + " / " denum[i] " равно " + numer[i] / denum[i]);
            }
            catch (ArithmeticException exc) {
                //Перехват исключения
                System.out.println("Попытка деления на нуль!");
            }
        }
    }
}

    Обработанное исключения удаляется из системы. На каждом шаге цикла блок try выполняется заново, а все возникшие ранее исключения считаются обработанными. Благодаря этому в программе могут обрабатываться повторно возникающие исключения.

    Выражения catch проверяются в том порядке, в котором они встречаются в программе.

    При множественном использовании инструкции catch, первым должен быть указан тип исключения, генерируемого подклассом, иначе вместе с исключением суперкласса будут перехвачены и все исключения подклассов, что вызовет ошибку.

    То есть, если планируется ловить исключение Throwable и ArrayIndexOutOfBoundException, то вервым стоит указывать ArrayIndexOutOfBoundException а затем уже Throwable. В таком случает Trowable будет перехватывать все исключения кроме ArrayIndexOutOfBoundException.

    Вложенные блоки try

    Исключение возникшее во внутрненнем блоке try и не перехваченное связанным с ним блоком catch распространяется во внешнйи блок try и обрабатывается связанным с ним блоком catch.

    Генерирование исключений (ключевое слово throw)

    Общая форма:
throw объект_исключения;

    Пример:
    try {
        //...
        throw new ArithmeticException();
    }

    catch (ArithmeticException exc) {
        //...
    }

    ППовторное генерирование исключений

    class Rethrow {
    public static void genException() {
    //...
    try {
        //...
    }
    catch (ArithmeticException exc){
        //...
    }
    catch (ArrayIndexOutOfBoundsException exc) {
        //...
        throw exc; //повторное генерирование исключения
    }
    }
}

class RethrowDemo {
    public static void main(String[] args) {
        try {
            Rethrow.genException();
        }
        catch (ArrayIndexOutOfBoundsException exc) {
            //Повторный перехват исключения
            //...
        }
    }
}  

    Ключевое слово finally

    Выполняется по завершении блока try/catch не зависимо от того, каким образом завершается блок try / catch.

    Форма записи:

try {
    //Блок кода, в котором отслеживаются ошибки
}
catch (тип_исключения_1 объект_исключения){
    //Обработчик исключения тип_исключения_1
}
catch (тип_исключения_2 объект_исключения) {
    //Обработчик исключения тип_исключения_2
}
//...
finally {
    //код блока finally
}

    Ключевое слово throws

    Общая форма:

возвращаемый_тип имя_метода (список_параметров) throws список_исключений {
    //Тело метода
}

список_исключений - разделенный запятыми список, в котором указываются исключения, генерируемые методом.

    Цепочные исключения

    Позволяет указывать одно исключение как причину появления другого (то есть найти истинную причину исключения).

    Конструкторы:   
    Throwable (Throwable причинное_исключение)
        - причинное_исключение - исключение, послужившее причиной текущего исключения
    Throwable (String сообщение, Throwable причинное_исключение)

    Эти же конструкторы введены в классы Error, Exception, RuntimeException.

    Методы:
    Throwable getCause()
         - возвращает исключение, которое стало причиное текущего исключения. Если такого не было - возвращает null.
    Throwable initCause(Throwable причинное_исключение)
         - связывает причинное исключение с исключением, которое должно быть сгенерировано, возвращая ссылку на него.
