Система ввода-вывода NIO

    Интерфейс Path

    Задает путь к файлу.

    Создание объекта типа Path не приводит к открытию  или созданию файла. Создается объект, инкапсултрующий путь к каталогу, в котором находится файл.

Метод
 Описание
boolean endsWith(String путь)
 Возвращает true, если вызывающий объект типа Path оканчивается заданным путем
boolean endsWith(Path путь)
 Возвращает true, если вызывающий объект типа Path оканчивается заданным путем
Path getFileName()
 Возвращает имя файла, связанное с вызывающим объектом типа Path
Path getName(int index)
 Возвращает объект типа Path содержащий имя элемента пути по указанному индексу в вызывающем объекте. Крайний слева соответствует элементу getNameCount() - 1
int getNameCount()
 Возвращает количество элементов(кроме корневого) в вызывающем объекте типа Path
Path getParent()
 Возвращает объект типа Path, который содержит весь путь, кроме имени файла, определяемого вызывающим объектом типа Path
Path getRoot()
 Возвращает корневой каталог
boolean isAbsolute()
 Возвращает true, если вызывающий объект обозначает абсолютный путь












    Экземпляр класса File(java.io) можно преобразовать в Path методом toPath(). Так же File (java.io) можно получить из Path методом toFile().

    Класс Files

    В основном состоит из статических методов, предоставляющие действия над файлами.
   
    Методы из класса Files используют объекты типа Path.

    Класс Paths

    Экземпляр типа Path нельзя создать непосредственно с помощью конструктора т.к. это интерфейс. Вместо этого можно получить объект типа Path с помощью метода get() в классе Paths.

        static Path get(String имя_пути, String части)
            Возвращает объект, инкапсулирующий определенный путь. Параметр "части" указывать не обязательно. Если путь не достоверен генерируется исключение  InvalidPathException.

        static Path get(URI uri)
            Возвращается объект типа Path, соответствующий заданному параметру uri.
