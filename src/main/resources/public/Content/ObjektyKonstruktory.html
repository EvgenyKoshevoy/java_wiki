Объекты, Конструкторы

    Объект это экземпляр класса.

    Процесс создания объекта:
    - выделяется память для хранения объекта;
    - создается ссылка на этот объект;
    - инициализируются переменные и вызывается конструктор.

    Инициализация объектов

Порядок инициализации переменных и вызова конструкторов:

  1. Инициализируются статические переменные базового класса
  2. Инициализируются статические переменные класса потомка
  3. Инициализируются нестатические переменные базового класса
  4. Начинает работу конструктор базового класса
  5. Инициализация нестатических полей дочернего класса
  6. Вызывается конструктор класса потомка
    Инициализация переменных класса происходит в конструкторе.
    Если не вызывается конструктор базового класса, его вызов будет добавлен автоматически.

    Пример:
Как все происходит на самом деле
class Pet {

Конструкторы

Конструктор инициализирует объект при его создании. Его имя совпадает с именем класса. Он похож на обычный метод, но у него нет возвращаемого типа.

Как правило, конструкторы используются для задания первоначальных значений переменных экземпляра, определенных в классе,
или же для выполнения любых других установочных процедур, которые требуются для создания полностью сформированного объекта.

Пример:

class MyClass {
    int x;
   
    MyClass() { //конструктор класса MyClass
        x = 10;
    }
}

Конструкторы можно перегружать, так же как и методы.

В  Java есть конструктор по умолчанию Он исчезает сразу после создания своего конструктора. Если нужен конструктор без параметор, его следует создать отдельно.
Пример:

public Cat() {
}

Параметризированные конструкторы

В конструктор параметры вводятся так же как и в метод.

Пример:
class Vehicle {
    int passengers;
    int fuelcap;
    int mpg;
   
    Vehicle (int p, int f, int m) {
        passengers = p;
        fuelcap = f;
        mpg = m;
    }
}

Инициализация объекта в коде:
Vehicle sportscar = new Vehicle (2, 14, 12);

В результате чего:
passengers = 2;
fuelcap = 14;
mpg = 12;
