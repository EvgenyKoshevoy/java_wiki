Интерфейсы

    Напоминает абстрактный класс, у которого все методы абстрактные.

    В интерфейсе указывается что именно следует делать, но не как это делать.

    Может быть реализован в любом колличестве классов. В одном классе может быть реализовано любое количество интерфейсов.

    Для реализации интерфейсов в классе должны быть предоставлены тела реализуемых методов, описанных в этом интерфейсе.

    Интерфейсы объявляются с помощью ключеового слова interface:

доступ interface имя {
    возвращаемый_тип имя_метода_1(список_параметров);
    возвращаемый_тип имя_метода_2(список_параметров);
    тип переменная_1 = значение;   
    тип переменная_2 = значение;
    //..
    возвращаемый_тип имя_метода_N(список_параметров);
    тип переменная_N = значение;
}

досуп - public, либо не указывается

    Методы объявленные в интерфейсы неявно считаются открытыми.

    Переменные объявленные в интерфейсе не являются переменными экземпляра. Неявно обозначаются ключевыми словами public, final и static (обязательно подлежат инициализации). По сути они являются константами.

    Пример:
public interface Series {
    int getNext();
    void reset();
    void setStart(int x);
}

Используется если:
    - ожидается, что несвязанные классы бубудт реализовывать интерфейс;
    - требуется определить поведение конкретного типа данных и не важно кто его реализует;
    - если хочешь использовать множественное наследование типа;

Чтобы поддержать реализацию интерфейса нужно:
    - унаследоваться от него;
    - реализовать объявленные в нем методы;
    - методы должны делать то, для чего они предназначены;

    Реализация интерфейсов

    Для реализации используется ключевое слово implements.

    Общая форма реализации:

class имя_класса extends супер_класс implements интерфейс {
    //телов= класса
}

    Если в классе реализовывается несколько интерфейсов, то их имена должны быть указаны через запятую. Extends указывать не бязательно.

    Реализуемые методы интерфейса должны быть объявлены открытыми (public).

    Пример:

class ByTwos implements Series {
    int start;
    int val;

    ByTwos(){
        start = 0;
        val = 0;
    }

    public int getNext() {
        val += 2;
        return val;
    }

    public reset() {
        start = 0;
        val = 0;
    }

    public void setStart(int x) {
        start = x;
        val = x;
    }
}

    Класс реализующий интерфейс может содержать дополнительные переменные и методы.

    Если класс реализует не все методы итерфейса, то он должен быть объявлен как abstract. Далее он может быть использован как суперкласс, а в его подклассах завершится раелиация методов.

    Применение интерфейных ссылок

    В Java можно объявлять переменные, хранящие ссылки на интерфейс. Она может ссылаться на любой объект, реализующий ее тип интерфейса.

    При вызове метода для объекта по интерфейсной ссылке выполняется варинт метода, реализованный в классе данного объекта.

    Переменная ссылки на интерфейс содержит информацию только о методах, объявленных в интерфейсе. Ее нельзя испольовать для доступа к переменным и методам, которые поддерживаются в этом объекте, но не объявлены в интерфейсе.

    Наследование интерфейсов

    Синтаксис наследования такой же как при наследовании классов. Может наследоваться только от интерфейсов. Интерфейсов родителей может быть много.
   
    Если класс реализует интрфейс наследующий другой, он должен реализовать все методы, объявленные в интерфейсах по всей цепочке наследования.

    Пример:

interface A {
    void meth1();
    void meth2();
}

interface B extends A {
    void meth3();
}

class MyClass implements B {
    public void meth1() {//...};

    public void meth2() {//...};

    public void meth3() {//...};
}

    Методы по умолчанию

    Объявление метода начинается с ключевого слова default. Данные методы реализуются в интерфейсе для испольованию по умолчанию. Этот метод не нужно реализовавыть в классе обязательно, он является опциональным.

    Класс может определить собственную реализацию метода.

    Пример:

public interface MyIF {
    int getUserID(); //Определение обычного метода

    default int getAdminID() {
        return 1;
    }
}

//метод по умолчанию не используется
class MyIFImp implements MyIF{
    public int getUserID() {
        return 100;
    }
}

//реализация обоих методов
class MyIFImp2 {
    public int getUserID() {
        return 100;
    }
   
    //Метод по умолчанию переопределен
    public int getAdminID() {
        return 46;
    }
}

    Множественное наследование

    Пример:
    Класс MyClass реализует два интерфейса: Alpha и Beta.
    Если оба интерфейса предоставляют метод reset() реализованный по умолчанию порядок реализации метода классом следующий:
    - приоритет имеет метод, переопределенный в подклассе;
    - если один интерфейс расширяет другой, то приоритет отдается наследющему интерфейсу;
    - если класс не переопределяет метод при использовании двух интеряейсов возникнет ошибка.

    На заданную по умолчанию реализацию можно ссылаться явно с помощью ключевого слова super:
    имя_интерфейса.super.имя_метода();

  Если из интерфейса Beta нужно обратиться к методу по умолчаию интерфейса Alpha используется следующая конструкция:
   
    Aplpha.super.reset();

    Статические методы интерфейса

    Вызов метода:
имя_интерфейса.имя_статического_метода();

    Пример:

public interface MyIF {
    static int getUniversalID() {
        return 0;
    }
}

    Пример вызова:
int uID = MyIF.getUniversalID();

    Для вызова метода не требуется реализовывать интерфейс или создавать экземпляр интерфейса. Статические методы интерфейса не наследуются и не реализуются классом или производными интерфейсами.

    Закрытые методы интерфейса

    Определяются с помощью ключевого слова private.
    Его невозможно использоват в коде, находящемся за пределами интерфейса, в котором этот метод определен. Данный метод не наследуется подчиненными интерфейсами.
