Основы ООП

Абстракиця - выделение наиболее важных характеритсик и информации об объекте.
Абстракцию так же можно определить, как способ представления элементов задачи из реального мира в виде объектов в программе.

Инкапсуляция - атрибуты и поведение объекта объединяются в одном классе, внутренняя реализация объекта скрывается от пользователя, а для работы с объектом предоставляется открытый интерфейс.
Задача программиста - определить какие атрибуты и методы будут доступны для открытого доступа, а какие являются внутренней реализацией объекта и дожны быть недоступны для изменений.

Состояние будущих объектов описывается в классе с помощью полей, а их поведение с помощью методов.
Возможность изменения состояния и поведения осуществляется с помощью модификаторов доступа к полям и методам - private, protected, public, default.

Пример:

public class SomePhone {
   
   private int year;
   private String company;
   public SomePhone (int year, String company) {
      this.year = year;
      this.company = company;
   }
   private void openConnection() {
   //find commutator
   //openNewConnection...
   }
   public void call(){
      openConnection();
      System.out.println("Вызываю номер");
}
   public void ring(){
      System.out.println("Дзынь дзынь");
}
}

Модификатор private делает доступными поля и методы класса только внутри этого класса. Это значит что получить доступ к полям из вне не возможно, так же как и получить доступ к private методам.

Сокрытие доступа с метода openConnetction дает нам возможность свободного изменения реализации этого метода, т.к. он гарантированно не используется другими объектами и его изменение не нарушит их работу.

Для работы с объектом мы оставляется открыте медоты ring и call с помощью модификатора public.
Предоставление открытых методов для работы с объектом так же является частью механизма инкапсуляции, т.к. если полностью закрыть доступ к объекту он становится бесполезным.

Наследование
В программировании наследовние заключается в использовании уже существующих классов для описания новых.

Пример создания класса смартфон с помощью наследования:

public abstract class AbstractPhone {
   private int year;
   
   public AbstractPhone (int year){
      this.year = year;
}
   public abstract void call (int outputNumber);
   public abstract void ring (int inputNumber);
}

public abstract class Wireless extends AbstractPhone {
   
   private int hour;
   public WirelessPhone (int year, int hour){
      super(year);
      this.hour = hour;
}
}

Полиморфизм - когда программа может использовать объекты с одинаковым интерфейсом без информации о внутреннем устройстве объекта.

public class User {
private String name;

public User(String name) {
this.name = name;
}

public void callAnotherUser(int number, AbstractPhone phone){
// вот он полиморфизм - использование в коде абстактного типа AbstractPhone phone!
phone.call(number);
}
}
}

Объект

 Объект (экземпляр) - это отдельный представитель класса, имеющий конкретное состояние и поведение, полностью определяемое классом.

Объект имеет конкретные значения атрибутов и методы, работающие с этими значениями на основе правил, заданных в классе.

Машина может быть красной или синей, кружко полной или пустой. Эти характеристики называются атрибутами объекта. Атрибут описывает текущее состояние объекта (еще его называют переменная экземпляра).

В ООП объект следующие свойства:
    - поведение объекта (что с ним можно делать, какие методы к нему можно применить)
    - состояние объекта (как объект реагирует на применение методов)
    - идентичность объекта (чем данный объект отличается от других, характеризующихся тем же поведением и состоянием)

Атрибут описывает текущее состояние объекта.
Поведение демонстрирует что объект способен делать.

Объект - значения переменных экземпляра могут отличаться от значений его полей ввода.
Объект и класс могут вести себя по разному.
Состояние объекта может меняться.
Объект может меняться во время выполнения программы.

Класс

Класс - способ описания сущности, определяющий состояние и поведение, зависящее от этого состояния, а так же правила для взаимодействия с этой сущностью (контракт). Другими словами, класс - описание объекта. Один класс можно использовать для создания множества объектов.

С точки зрения программирования класс можно рассматривать как набор данных (полей, атрибутов, членов класса) и функций для работы с ними (методов).

С точки зрения структуры программы класс является сложным типом данных.

Каждый класс имеет имя, которое используется для определения атрибутов и поведения.

Атрибуты: имя, высота, вес, пол, возраст.
Поведение: ходить, бегать, прыгать, говорить, спать.

Класс - компилируется из файла Java.
Класс объявляет методы.

Методы


Методы определяют поведение.
Метод является коллекцией выражений, сгруппированных вместе для выполнения действия.
System.out.println() - является примером метода.

Метод и переменная экземпляра находятся в объектах.

Можно определить свои методы для выполнения желаемых действий.

class MyClass {

static void sayHello() {
System.out.println("Hello World!");
 }

public static void main(String[] args) {
   sayHello();
   }
}

Чтобы вызвать метод, достаточно ввести его имя и пару круглых скобок.

Вызов методов

Метод можно вызывать любое количество раз. При запуске метода код перескакивает в место, где метод был определен, выполняет код внутри него, а затем возвращается обратно и выполняет другие строки.

Параметры методов

Можно создать метод, который принимает некторую информацию, называемую параметрами, при его вызове. Параметры необходимо указывать внутри скобок.
Например можно модифицировать метод sayHello(), чтобы он принимал параметр String.

myClass {
   static void sayHello(String name) {
      Systwm.out.println("Hello" + name);
   }

public static void main(String[] args) {
   sayHello("David");
   sayHello("Amy");
   }
}
Метод sayHello принимает строку с именем name в качестве параметра, который используется в теле метода. При вызове метода мы передаем значение параметра в скобки. 

Метод может иметь множество параметров, разделенных запятыми.

Преимущество методов - повторное использование кода. Можно написать метод один раз и использовать его множество раз. Основываясь на переданных параметрах, метода могут выполнять различные действия.

Интерфейс

Интерфейс - набор методов класса, доступных для использования другими классами.

Интерфейсом класса будет являться набор всех его публичных методов в совокупности с набором публичных атрибутов. Интерфейс специфизирует класс, четко определяя все возможные действия над ним.
