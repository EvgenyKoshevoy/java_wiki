Ключевые слова

Ключевое слово static

Доступ к переменной static осуществляется напрямую, без создания объекта и обращения через него к экземпляру класса.

Пример статической переменной:

class StaticDemo {
    int x; //обычная переменная экземпляра
    static int val; //статическая переменная
    //..

    }

class SDemo {
    public static void main(String[] args){

        StaticDemo ob1 = new StaticDemo();
        StaticDemo.y = 19; //обращение к статической переменной

    }
}

Ограничения на применение методов типа static:

  • в методе типа static допускается непосредственный вызов только других методов static;
  • для метода типа static непосредственно доступными оказываются только другие данные типа static, определенные в его классе;
  • в методе типа static должна отсутствовать ссылка this;
Пример некорректно созданного статического метода:

class StaticError {
    int denom = 3;
    static int val = 1024;

    //ошибка! К нестатическим переменным нельзя обращаться из статического метода.
    static int valDivDenom() {
        return val / denom; //не пройдет компиляцию
    }
}

Статические блоки

Статический блок выполняется при первой загрузке класса, еще до того как класс будет использован.

Пример:

class StaticBlock {
    static double rootOf2;
    static double rootOf3;

    static {                                     //этот блок выполнится при загрузке класса
        rootOf2 = Math.sqrt(2.0);
        rootOf3 = Math.sqrt(3.0);

    }

}

Ключевое слово this

Ключевое слово this обозначает именно тот объект, по ссылке на которой действует вызываемый метод.

Важно запомнить:
Java не запрещает использовать имена параметров или локальных переменных, совпадающие с глобальными переменными. В этом случае говорят, что локальная переменная или параметр скрывает переменную экземпляра. Доступ к скрытой переменной экземпляра обеспечивается ключевым словом this.

Пример конструктора синтаксически правилен, но такого стиля программирования стоит избежать:

Pwr (int b, int e) {
    this.b = b;
    this.e = e;
    //..
}

Ключевое слово super

1. Используется для вызова конструктора суперкласса из подкласса.
Пример:

class TwoDShape {
    double width;
    double height;
    //...
    TwoDShape(double w, double h) {
        width = w;
        hieght = h;
    }
}

class Triangle extends TwoDShape {
    String style;
    //...
    Triangle (String s, double w, double h) {
    super (w, h); //вызов метода конструктора TwoDShape
    style = s;
    }
}

2. Используется для доступа к членам суперкласса

Синаксис:
super.член_класса;

Пример:
class A {
     int i;
}

class B extends A {
    int i;
    B (int a, int b) {
        super.i = a; //ссылка на переменную класса А
        i = b; //переменная класса B
    }
}

Ключевое слово final

Используется для запрета переопределения метода или наследования класса.

Предотвращение переопределения методов

Для предотвращения переопределния в начале объявления метода нужно указать ключевое слово final.

Пример:

    class A {
        final void meth() {
            System.out.println("Это метод final.");
        }
    }

    class B extends A {
        void meth() { //Ошибка: этот метод не может быть переопределен!
            System.out.println("Недопустимо!");
        }
    }

Предотвращение наследования

    Предотвратить наследование класса можно, указав в определении класса ключевое слово final. В этом случает считается, что это ключевое слово применяется ко всем методам класса. Применять его к абстрактным классам не имеет никакого смысла, т.к. абстрактный класс требует переопределения методов.

Пример:

     final class A {
        // ...
    }

    //Следующее определение класса недопустимо
    class B extends A { //Ошибка: класс А не может иметь подклассов!
        // ...
    }

Применение к переменным экземпляра

    Используется для создания именованных констант. Если имени переменной предшествует final, то ее значение нельзя изменить до конца работы программы.

    Название переменных пишется КАПСОМ, а между словами ставится нижнее подчеркивание.
