Обобщения

    Общая форма обобщенного класса:
        class имя_класса<список_параметров_типа> {// ...}

    Синтаксис объявления ссылки на обобщенный класс:
        имя_класса<список аргументов_типа> имя переменной = new имя_класса<список_аргументов_типа>(список_аргументов_конструктора);     

// простой обобщенный класс.
// Здесь Т - это параметр типа, вместо которого при создании объекта класса Gen будет подставляться
// реально существующий тип.

class Gen<T> {
    T ob; // объявить объект типа Т
   
    //передать конструктору ссылку на объект типа Т
    Gen(T o) {
        ob = o;
    }
   
    // вернуть объект ob из метода
    T getOb() {
        return ob;
    }
   
    // отобразить  тип Т
    void showType() {
        System.out.println("Тип Т - это " + ob.getClass().getName());
    }
}		 // демонстрация использования обобщенного класса
public class GenDemo {
    public static void main(String[] args) {
        // создать обобщенную ссылку на целочисленное значение
        Gen<Integer> iOb;
        iOb = new Gen<Integer>(88);
       
        iOb.showType();
       
        int v = iOb.getOb();
       
        System.out.println("значение: " + v);
        System.out.println();
       
        // создать объект типа Gen для строк
        Gen<String> strOb;
        strOb = new Gen<String>("Testing generics");
       
        strOb.showType();
       
        String str = strOb.getOb();
        System.out.println("Значение: " + str);
    }
}


    Обобщения работают только с объектами. Простые типы не могут выступать в роли передаваемого аргумента.

    Обобщенный класс с двумя параметрами:

// простой обобщенный класс с двумя параметрами T и V
class TwoGen<T, V> {
    T ob1;
    V ob2;
   
    TwoGen(T ob1, V ob2) {
        this.ob1 = ob1;
        this.ob2 = ob2;
    }
   
    // отобразить типы T и V
    void showTypes() {
        System.out.println("Тип Т - это " + ob1.getClass().getName());
        System.out.println("Тип V - это " + ob2.getClass().getName());
    }
   
    T getOb1() {
        return ob1;
    }
   
    V getOb2() {
        return ob2;
    }
}
 // демонстрация класса TwoGen
public class SimpGen {
    public static void main(String[] args) {
        TwoGen<Integer, String> tgObj = new TwoGen<Integer, String>(88, "Generics");
       
        // отобразить типы
        tgObj.showTypes();
       
        // получить и отобразить значения
        int v = tgObj.getOb1();
        System.out.println("значение: " + v);
       
        String str = tgObj.getOb2();
        System.out.println("значение: " + str);
    }
}


    Органиченные типы

    Java позволяет создавать обощения с ограниченными типами. Это необходимо, если например требуется работать только с числовыми данными.

    Объявление:
        <T extends суперкласс>
        class NumericFns<T extends Number>

    Также есть возможность обеспечивать совметстимость одного параметра с другим:
        class Pair<T, V extends T> {
            T first;
            V second;

            Pair(T first,V second) {
                this.first = first;
                this.second = second;
            }
            // ...
        }

    Шаблоны аргументов

    Обозначается метасимволом ?, которому соответствует неизвестный тип данных.

    Пример:
    class NumericFns<T extends Number> {
        // ...

        boolean absEquals(NumericFns<?> ob) {    // ? - соответствует любому типу объекта из класса NumericFns.
            // ...
        }
    }

    Ограниченные шаблоны аргументов

    Задача: UseBoundedWildcard

    Шаблоны можно ограничивать так же, как и параметры типов.  Ограничение полезно при создании методов, которые должны оперировать только объектами подклассов определенного суперкласса.

    Для настройки верхней границы шаблона аргумента используется следующее выражение:
        <? extends суперкласс> - суперкласс - определяет верхнюю границу, включая его самого. В качестве аргумента допускается указывать не только подклассы но и сам суперкласс.

    Для настройки нижней границы используется следующее выражение:
        <? super подкласс> - в качестве аргумента допускается использовать только суперклассы, от которых наследуется подкласс, включая его самого.  

    Пример:
        static void test(Gen<? extends A>) {// ...}    // в качестве аргумента могут выступать все подклассы А, включая его самого.


    Обобщенные методы

    Задача: GenericMethodDemo

    Методы в обобщенных классах могут использовать параметр типа своего класса и автоматически становятся обобщенными относительно параметра типа. Так же можно объявить метод, который сам по себе использует параметр типа. Такой метод может быть объявлен в обычном, а не в обобщенном классе.

     Форма объявления метода:
         <списко_параметров_типа> возвращаемый_тип имя_метода(список параметров) {// ...}

    Пример:
    static <T extends Comparable<T>, V extends T> boolean arraysEqual(T[] x, V[] y) {// ...}

    Обобщенные конструкторы

    Задача: GenConsDemo

    Конструктор может быть обобщенным, даже если сам класс не явяется обобщенным.

    Пример:
        <T extends Number> Summation(T arg) {// ...}

    Обобщенные интерфейсы

    Задача: GenIFDemo

    Синтаксис:
        interface имя_интерфейса<список_параметров_типа> {//... }

    Общая форма объявления класса, реализующего обобщенный интерфейс:
        class имя_класса<список_параметров_типа> implements имя_интерфейса<список_параметров_типа> {// ...}

    Пример:
        interface Containment<T> {//... }

        class MyClass<T> implements Containment<T> {//... }

    Если верхняя граница параметра задана в объявлении класса, то нет необходимости указывать ее еще раз после implements:
        class MyClass<T extends Number> implemets Containmet<T>

    Базовые типы

    Совместимы с унаслежованным кодом. Не обеспечивается безопасноть типов.

    Пример:
    class Gen<T> {
        T ob;     // объявить объект типа Т

        // передать конструктору ссылку на объект типа Т
        Gen(T o) {
            ob = o;
        }

        // вернуть объект ob
        T getob() {
            return ob;
        }
    }

    Выведение типов

    С JDK 7 для создания экземпляров обобщенного типа предусмотрен сокращенный синтаксис:
        имя_класса<список_аргументов_типа> имя_переменной = new имя_класса<>(список_аргументов_конструктора);
        где: <> - ромбовидный оператор, обозначающий пустой список аргументов типа.

    Пример:
        TwoGen<Integer, String> tgOb = new TwoGen<>(42, "testing");

    Ошибки неоднозначности

    Пример:
// неоднозначность, вызванная очисткой перегруженных методов
class MyGenClass<T, V> {
    T ob1;
    T ob2;
   
    // ...
   
    // эти два объявления перегруженных методов порождают неоднозначность
    // код не скомпелируется
    void set(T o) {
        ob1 = o;
    }
   
    void set(T o) {
        ob2 = o;
    }
}

    Для комплятора эти методы одинаковы:
        1: Можно создать объект <String, String>
        2: После очистки происходить приведение типов: void set(Object o)

    В таких случаях требуется задавать разные имена методов.

    Ограничения на использование обобщений

    Невозможность создания экземпляров параметров типа

    Создать экземпляр параметра типа невозможно: компилятору ничего не известно о типе создаваемого объекта. Тип Т явялется заполнителем, информация о котором удаляется во время компиляции.

    Пример:
    //невозможно получить экземпляр типа Т
    class Get<T> {
        T ob;

        Gen() {
            ob = new T();    //недопустимо
        }
    }

    Ограничения статических членов класса

    В статическом члене нельзя использовать параметры типа, объявленные в его классе.

    Пример:
    class Wrong<T> {
        //неверно, поскольку нельзя создать статическую переменную типа Т
        static T ob;

        // неверно, поскольку невозможно использовать переменную типа Т в статическом методе
        static T getOb() {
            return ob;
        }
    }

    Ограничения обобщенных массивов

    Нельзя получить экземпляр массива, тип элементов которого определяется параметром типа.

    Нельзя создать массив обобщенных ссылок на объекты конкретного типа.

    Пример:
    class Gen<T extends Number> {
        T ob;

        T[] vals;    //допустимо
       
        Gen(T o, T[] nums) {
            ob = o;

            //недопустимо
            vals = new T[10];

            //допустимо
            vals = nums;
        }
    }
    
    class GenArrays {
        public static void main(String[] args) {
            Integer[] n = {1, 2, 3, 4 ,5};

            Gen<Integer> iOb = new Gen<Integer>(50, n);

            //невозможно создать массив обобщенных ссылок на объекты конкретного типа
            Gen<Integer> gens[] = new Gen<Integer>[10];

            // допустимо
            Gen<?> gens[] = new Gen<?>[10];
        }
    }

    Ограничения обобщенных исключений

    Обобщенный класс не может расширять класс Throwable. Создать обобщенные классы исключений невозможно.
