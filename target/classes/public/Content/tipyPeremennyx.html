Всего 4 группы переменных:

  1. Целочисленные (byte, short, int, long)
  2. С плавающей точкой (float, double)
  3. Символы (char)
  4. Логические (boolean)

Эти 8 типов переменных относятся к примитивным типам переменых.

Значения переменных типа float должны применяться с буквой f. Пример: float a = 32.5f; Без этой буквы все значения с точкой в Java относятся к типу double.

Нельзя поместить большое значение в малое.
Пример:

int x = 24;
byte b = x;

Эта операция не сработает, т.к. тип int занимает больше памяти, чем byte. 4 байта в 1 поместить не получится.
Обратная операция возможна.

Прежде чем использовать переменную, ее необходимо объявить.

5 правил выбора названий переменных:

  1. Переменные пишутся только латинскими буквами.
  2. Имя переменной, по возможности должно быть "говорящим".
  3. Если переменная состоит из 2 и более слов, пишется в сamelStyle.
  4. Есть 54 слова, которые нельзя использовать в названии переменных (boolean, byte, char, double, float, int, long, short, public, private, protected, abstract, final, native, static, strictfp, synchronized, transient, volatile, if, else, do, while, switch, case, default, for, break, continue, assert, class, extends, implements, import, instanceof, interface, new, package, super, this, catch, finally, try, throw, throws, return, void, const, goto, enum).
  5. Имя переменной:
    • может начинаться с любых латинских букв, $ или _
    • не может начинаться с цифр

    Приведение типов

    Есть два вида приведения типов: расширение и сужение.
   
    char равен short но свободно приводить их нельзя:
    - при преобразовании из short в char всегда будут теряться значения меньше 0;
    - при преобразовании из char в short будут теряться значения больше 32000;
   
    При преобразовании из целых в дробные могут теряться самые младшие части числа.

    При сужении диапазона используется явное приведение:
        int c = 10000;
        byte b = (byte) c;

    Оператор приведения действует только на переменную/число, идущее прямо за ним:
    float f = 10000;
    long l = (float) f * f; //не скомпилируется т.к. умножение long на float дает float.
    Правильный способ приведения:
    float = 10000;
    long l = (float) (f * f);

Литералы

Литералы это конкретные значения переменных, записанные прямо в коде.
    123456 - int
    222333444422L - long
    12.323254f - float
    12.333333333333333d - double
    "Mama", "" - String
    '\u3232', 'T' - char
    true, false - boolean
    null - Object

В литералы можно вставлять знак подчеркивания, для удобства восприятия числовых значений, состоящих из нескольких чисел.
Например:123_45_1234.

Литералы можно представлять в шестнадцатиричном, восьмеричном и двоичном видах.
hex = 0xff; //для представления в шестнадцатиричном виде вначале числа ставится 0x
oct = 011; //для представления в восьмерином в начале ставится 0
binar = 0b11000; //для представления в двоичной форме в начале ставится 0b

Управляющие последовательности символов

Некоторые знаки или непечатные симолы имеют специальное назначение, поэтому их нельзя указывать в качестве литерала. Поэтому, в Java предусмотрены управляющие последовательности, начинающиеся с обратной косой черты (еще называют экранирование символа).

\' - одинарная ковычка
\" - двойная ковычка
\\ - обратная косая черта
\r - возврат коретки
\n - перевод строки
\f - перевод страницы
\t - горизонтальная табуляция
\b - возврат на одну позицию
\ddd - восьмеричная константа, где ddd - восьмеричное число
\uxxxx - шестнадцатиричная константа, где xxxx - шестнадцатиричное число

Область действия переменных

Блок определяет область действия переменной (блок начинается с открытой фигурной скобки, а заканчивается закрывающей фигурной скобкой).
Начиная новый блок,  мы создаем новую область действия.

Как правило, переменные, объявленные в некоторой области действия, невидимы (а следовательно, недоступны) за ее пределами.
Правила определения области действия служат основанием для инкапсуляции.

Области действия мoryr быть вложенными. Открывая новый блок кода, вы создаете новую, вложенную область действия. Такая область заключена во внешней области. Это означает, что объекты, объявленные во внешней области действия, будуr доступны для кода во внутренней области, но не наоборот.

Переменные можно объявлять в любом месте блока кода, но сделать это следует непременно перед тем, как пользоваться ими.

Следует также иметь в виду, что переменные, созданные в области их действия, удаляются, как только управление в программе передается за пределы этой области.

Для правил области действия в Java характерна следующая особенность: во вложенном блоке нельзя объявлять переменную, имя которой совпадает с именем переменной во внешнем блоке.

Объектные переменные

На самом деле, не существует такого понятия как объектная переменная, есть переменная, ссылающаяся на объект. Она хранит биты, которые описывают путь для доступа к объекту.

Переменная хранит не объект а что-то вроде указателя или адреса. Что на самом деле хранится внутри этой переменной неизвестно. Известно, что она представляет лишь один объект и JVM умеет использовать ссылку для его получения.

Нельзя вложить объект в переменную, т.к. не существует таких больших контейнеров для хранения объекта.

Ссылочные переменные содержат биты, которые описывают способ получения объекта.

Dog d = new Dog();
d.bark(); - стоит трактовать как "Используй элемент перед точкой, чтобы дать мне элемент, находящийся после точки. Т.е. используй объект, на который ссылается переменная d , чтобы вызвать метод bark().

Dog myDog = new Dog();

1. Объявление ссылочной переменной
Dog myDog = new Dog();
Это говорит JVM, что она должна выделить место в памяти для ссылочной переменной и назвать ее myDog. Ссылочная переменная навсегда получает тип Dog.

2. Создание объекта:
Dog myDog = new Dog();
JVM получает приказ выделить память в куче для объекта Dog.

3. Связывает объект и ссылку:
Dog myDog = new Dog();
Присваивает ссылочной переменной myDog новый объект типа Dog.

Взаимоотношения ссылок и объектов


Book b = new Book();
Book c = new Book();

Объявляем две ссылки типа Book, создаем два объекта Book. Присваиваем ссылочным перемеменным объекты типа Book.
Ссылок 2, объектов 2.

Book d = c;

Объявляем новую ссылку типа Book. Вместо того, чтобы создавать новый третий объект, присываем ссылке значение с (возьми биты из с и помести копию в d).
Переменные c и d ссылаются на один объект. Они хранят две разные копии одного и того же значения.
Ссылок 3, объектов 2.

c = b;

Значение ссылки b присваивается переменной с.
с и b ссылаются на один объект.

Преобразование ссылочных типов

Переменной типа Object можно присвоить ссылку любого типа (это будет расширением типа).
Обратное присваивание нужно указывать явно.
   
    String s = "mama";
    Object o = s; //расширение

    Object o = "mama";
    String s = (String) o; //сужение

Как работает сборщик мусора?


Book b = new Book();
Book c = new Book();
b = c; //теперь b и c ссылаются на один объект. Объект, который принадлежал b становится недоступным и пригоден для сборщика мусора.
c = null; //Это превращает ее в нулевую ссылку и теперь она ни на что не ссылается. Ей все еще можно присвоить другой объект типа Book.
