Преобразование типов Java

    Примитивные типы:

    Автоматическое преобразование

Каждый тип переменных имеет свой диаппазон значений и объем занимаемой памяти.

Сравним:

1. byte и short. byte имеет меньший диапазон допустимых значений, чем short. Значит, byte можно вложить в short.

2. byte и int. byte имеет меньший диапазон допустимых значений, чем int. Значит, byte можно вложить в int.

3. int и long. int имеет меньший диапазон допкстимых значений, чем long. Значит, int можно вложить в long.

Это пример автоматического преобразования. Схематически это выглядит так:

Пример №1

Код №1. Ответ: 15

     class Test {
 
	     public static void main(String[] args) {
		     byte a = 15;
		     byte b = a;
		     System.out.println(b);
	     }
     }

Код №2. Ответ: 15

     class Test {
 
	     public static void main(String[] args) {
		     byte a = 15;
		     int b = a;
		     System.out.println(b);
	     }
     }

Код №1 отличается от кода №2. В коде №2 присутствует автоматическое преобразование типов, а в коде №1 нет.

Число одно и то же, но оно теперь находится в большем конейнере, которое занимает болоьше места.

    Приведение типов

Может быть проблемпным перемещение из большего контейнера в меньший. Если пользователю известно, что помещаемое число входит в границы нового контейнера, Java об этом не знает. Ей нужно на это указать явно.

     class Test {
	     public static void main(String[] args) {
		     int a=0;
		     long b=15;
		     a = (int) b;
	     }
     }

Тут дописали (int) перед b. Если бы переменная a была бы типа byte, к примеру, то в скобах было бы (byte).


Она говорит "сделай мне из (большего) значения b переменную нужного мне (целевого) типа int.

    Если что-то пошло не так:

Если в контейнер поместить то, что туда не помещается - в него поместиться только то, что туда влезло. Например, у числе с плавающей точкой будет отсекаться дробная часть. Надо помнить, что дробная часть не округляется, а отбрасывается

Если поместить число, выходящее за границы контейнера, например, если в byte (-128; 127) поместить число 128 - получим -128.

Цель программиста не допускать выхода значения за допустимые границы, поскольку это может привести к неправильной работе программы.

     Итоги:

  1. При приведении float или double к целочисленным типам, дробная часть не округляется, а отбрасывается.

  2. Тип boolean не приводится ни к одному из типов.

  3. Тип char приводится к числовым типам как символ в системе Unicode.

  4. Если число больше своего контейнера результат будет непредсказуемым.

    Ссылочные типы:

    Цепочка наследования:
Класс, его родитель, родитель родителя, ... , Obejct.

    Т.к. класс содержит все методы класса-родителя, то объект этого класса можно сохранить в переменную любого из его типов родителей.

class Animal {
public void doAnimalActions();
}
class Cat extends Animal {
public void doCatActions();
}
class Tiger extends Cat {
public void doTigerActions();
}

public static void main(String[] args) {
Tiger tiger = new Tiger();
Cat cat = new Tiger();
Animal animal = new Tiger();
Object obj = new Tiger();
}

    При таком присваивании с объектом ничего не происходит. Меняется только количество методов, которое можно вызвать с помощью конкретной переменной-ссылки.

    Сужение типа:

    При сужении типа используется явное проеобразование:

public static void main(String[] args) {
Object obj = new Tiger();
Animal animal = (Animal) obj;
Cat cat = (Cat) obj;
Tiger tiger = (Tiger) animal;
Tiger tiger2 = (Tiger) cat;
}

Чтобы избежать ошибок при сужении типов, рекомендуется исяпользовать проверку instanceOf:

public static void main(String[] args) {
    Animal animal = new Tiger();
    doAllAction(animal);

    Animal animal2 = new Cat();
    doAllAction(animal2);

    Animal animal3 = new Animal();
    doAllAction(Animal3);

    public static void doAllAction (Animal animal) {
        if (animal instanceof Tiger) {
            Tiger tiger = (Tiger) animal;
            tiger.doTigerActions();
        }
        if (animal instanceof Cat) {
            Cat cat = (Cat) animal;
            cat.doCatActions();
        }

        animal.doAnimalActions();
    }
}

    Расширение типа:

    Движение вверх по цепочке называется расширением типа. При этом теряется возможность использовать методы, добавленные в класс при наследовании.

public static void main(String[] args) {
Cat cat = new Tiger();
Animal animal = cat;
Object obj = cat;
}

    При расширении типа оператор преобразования не нужен, т.к. не требуется проверка. Ссылку на объект всегда можно сохранить в переменную любого его базового типа.
